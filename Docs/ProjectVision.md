# Project Vision

# Incantation

## Vision

Incantation is a competitive multiplayer party game set in a dark fantasy universe.

Players gather around an ancient cursed table, each carrying a magical spellbook capable of casting powerful spells through spoken incantations.

The objective is not only to win.

The objective is to create unforgettable moments between friends.

Every match should generate laughter, tension, betrayal, unexpected twists and memorable stories that players will remember long after the game ends.

Dark fantasy and horror are used to amplify these moments, not replace them.

---

# Mission

Create the most memorable voice-controlled multiplayer party game ever made.

Every mechanic should encourage interaction between players rather than isolated gameplay.

---

# Core Philosophy

Fun always comes first.

If a mechanic is realistic but not fun, it should be redesigned.

If a mechanic creates unforgettable stories, it belongs in Incantation.

---

# Pillars

## 1. Voice is Magic

Players cast spells by speaking forbidden words.

The player's voice is their primary weapon.

---

## 2. Every Match Creates a Story

No two games should ever feel the same.

Players should leave every session with new stories to tell.

---

## 3. Betrayal Creates Memories

Unexpected betrayals, deception and clever plays create the strongest emotions.

---

## 4. Players Create the Experience

The game provides the tools.

The players create the chaos.

---

## 5. Fun Before Realism

Gameplay always takes priority over realism.

The goal is memorable experiences.

---

# Game Modes

## Last Mage Standing

Competitive multiplayer.

The last surviving mage wins.

Players curse, sabotage and deceive each other using spoken magic.

---

## Campaign

Cooperative multiplayer.

Players work together against supernatural forces, complete rituals, survive haunted locations and uncover the secrets behind the cursed books.

---

# Design Rule

Every new feature must answer one question:

"Will players remember this moment tomorrow?"

If the answer is no, the feature should be reconsidered.
